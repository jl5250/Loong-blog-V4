## Issues Found

### ISSUE-001 — MusicPlayer onEnded stale closure
**Severity:** Medium | **Category:** Functional

The audio `onEnded` handler uses `tracks` and `currentMusic` from closure but the `useEffect` deps only list `[track?.id, track?.url]`. If playlist changes during playback, next-track selection could use stale data.

**File:** `src/components/music/MusicPlayer.tsx:110-115`

### ISSUE-002 — Missing alt attributes on images
**Severity:** Low | **Category:** Accessibility

6 images have empty `alt=""` instead of meaningful descriptions (MusicPlayer track cover, comment avatars, FootprintMap markers, resume avatar). This hurts screen reader users.

### ISSUE-003 — next.config.ts remotePatterns incomplete
**Severity:** Low | **Category:** Config

Only allows `thrivex.**` and `localhost:8080`. Actual images load from `bu.dusays.com`, `q1.qlogo.cn`, `bucket.loongai.fun`, `res.liuyuyang.net`. Not breaking (app uses `<img>` not `next/image`) but will bite on migration.

### ISSUE-004 — Empty catch blocks swallow API errors
**Severity:** Low | **Category:** UX

`Header.tsx:32,38` silently catches `getCateList()` and `getThemeConfig()` failures. If backend is down, nav is empty with no user feedback.

### ISSUE-005 — Footer nav links are static
**Severity:** Cosmetic | **Category:** Content

Footer has 6 hardcoded links but misses routes like 数据、设备、说说、留言墙.
